# Лекция: Bash-программирование

---

## 1. Зачем bash-скрипты?

Bash — не универсальный язык программирования. У него есть чёткая ниша:

| Инструмент | Ниша |
|---|---|
| `grep` / `sed` / `awk` | Обработка текста, преобразование данных |
| Python / Go / Java | Бизнес-логика, сложные алгоритмы, работа с данными |
| **Bash** | **Склейка программ**: запустить несколько утилит, проверить результаты, принять решение |

Типичные задачи для bash-скриптов:
- Интеграционные тесты: поднять несколько компонентов и проверить, что они работают вместе.
- Автоматизация повторяющихся действий в командной строке.
- CI/CD-пайплайны: сборка, тесты, деплой.
- Инфраструктурные скрипты: создать структуру каталогов, скачать файлы, настроить окружение.

Bash **неудобен** для: чистой арифметики, работы со структурами данных, сложной обработки строк.

## 2. Переменные

### Присваивание

В bash присваивание должно быть **без пробелов** вокруг `=`:

```bash
name="Alice"       # правильно
name = "Alice"     # ОШИБКА: bash воспримет "name" как команду
```

Обращение к переменной — через `$`:

```bash
echo $name
echo "Hello, $name"
```

### Переменные окружения

Переменные, установленные в текущей оболочке, не видны дочерним процессам. Чтобы передать переменную дочерним процессам, нужна команда `export`:

```bash
export MY_VAR="value"
```

Стандартные переменные окружения:

| Переменная | Значение |
|---|---|
| `$HOME` | Домашний каталог |
| `$PATH` | Каталоги поиска исполняемых файлов |
| `$PWD` | Текущий рабочий каталог |
| `$USER` | Имя текущего пользователя |

## 3. Кавычки

В bash три способа экранирования:

### Одинарные кавычки `'...'`

Отключают **все** специальные символы. Переменные внутри **не подставляются**:

```bash
echo '$HOME'     # выведет: $HOME (буквально)
echo 'Hello $USER'  # выведет: Hello $USER
```

### Двойные кавычки `"..."`

Отключают большинство специальных символов, но **переменные и подстановки подставляются**:

```bash
echo "$HOME"        # выведет: /home/vasya
echo "Hello $USER"  # выведет: Hello vasya
```

Двойные кавычки **обязательны**, когда значение переменной может содержать пробелы:

```bash
name="my dir"
mkdir $name         # ОШИБКА: создаст два каталога "my" и "dir"
mkdir "$name"       # правильно: создаст один каталог "my dir"
```

### Обратный слэш `\`

Экранирует один следующий символ:

```bash
mkdir my\ dir       # создаст каталог с пробелом
echo \$HOME         # выведет: $HOME буквально
```

## 4. Параметры скрипта

При запуске скрипта аргументы командной строки доступны через специальные переменные:

| Переменная | Значение |
|---|---|
| `$0` | Имя скрипта |
| `$1`, `$2`, ... | Первый, второй, ... аргумент |
| `$#` | Количество аргументов |
| `$*` | Все аргументы одной строкой |
| `$@` | Все аргументы по отдельности (предпочтительнее) |
| `$?` | Код завершения последней команды |

**Пример скрипта:**

```bash
#!/bin/bash
# Создать архив из каталога, переданного как аргумент

name="$1"    # лучше сразу дать переменной смысловое имя

tar -czf "${name}.tar.gz" "$name"
```

Запуск: `./make_archive.sh mydir`

## 5. Команда test и условные выражения

### Команда `test`

Проверяет условие и возвращает код 0 (true) или 1 (false):

```bash
test "$#" -eq 0     # количество аргументов равно 0?
test -f file.txt    # файл существует?
test -d /tmp/mydir  # каталог существует?
test -n "$str"      # строка не пуста?
test -z "$str"      # строка пуста?
```

### Альтернативный синтаксис `[...]`

`[ условие ]` — это синоним `test`, чаще используется в скриптах:

```bash
[ "$#" -eq 0 ]
[ -f file.txt ]
[ "$a" = "$b" ]      # строковое равенство
[ "$a" -eq "$b" ]    # числовое равенство
[ "$a" -gt "$b" ]    # числовое "больше"
[ "$a" -lt "$b" ]    # числовое "меньше"
```

**Важно:** пробелы внутри скобок обязательны: `[ -f file ]`, не `[-f file]`.

### Проверка типов файлов

| Флаг | Проверка |
|---|---|
| `-f` | Обычный файл |
| `-d` | Каталог |
| `-e` | Файл/каталог существует |
| `-r` | Есть право на чтение |
| `-w` | Есть право на запись |
| `-x` | Есть право на выполнение |
| `-s` | Файл не пуст |

## 6. Условная конструкция if

### Синтаксис

```bash
if условие; then
    действие_если_true
elif другое_условие; then
    действие
else
    действие_если_false
fi
```

В bash после `if` и `elif` стоит **команда** (а не выражение). Команда выполняется, и если она вернула 0 — условие истинно.

### Пример: проверка аргументов

```bash
#!/bin/bash

if [ "$#" -eq 0 ]; then
    echo "Usage: $0 <dirname>" >&2
    exit 1
fi

name="$1"
tar -czf "${name}.tar.gz" "$name"
```

### Логические операторы вместо if (для простых случаев)

```bash
# Если нет аргументов — вывести ошибку и выйти
[ "$#" -eq 0 ] && echo "No arguments" >&2 && exit 1

# Создать каталог, если его нет (подавить ошибку если уже есть)
mkdir -p "$1" 2>/dev/null

# Запустить что-то, если предыдущая команда успешна
make && ./run_tests

# Fallback: попробовать второе, если первое упало
command_a || command_b
```

## 7. Циклы

### Цикл `for`

```bash
# По явному списку
for item in a b c 1 2 3; do
    echo "$item"
done

# По аргументам скрипта
for arg in "$@"; do
    echo "Аргумент: $arg"
done
```

**Практический пример** — создать архив из каждого переданного каталога:

```bash
#!/bin/bash
for name in "$@"; do
    tar -czf "${name}.tar.gz" "$name"
done
```

### Цикл `while`

```bash
while условие; do
    тело_цикла
done
```

Пример — разбор аргументов командной строки:

```bash
while [ "$#" -gt 0 ]; do
    case "$1" in
        --outdir)
            outdir="$2"
            shift 2
            ;;
        --indir)
            indir="$2"
            shift 2
            ;;
        -*)
            echo "Unknown option: $1" >&2
            exit 1
            ;;
        *)
            break   # началась часть с файлами (не опциями)
            ;;
    esac
done
```

## 8. Команда `shift`

`shift` сдвигает все позиционные параметры: `$2` становится `$1`, `$3` становится `$2` и т.д. Значение `$1` удаляется. `$#` уменьшается на 1.

```bash
# До shift: $1=--outdir $2=/tmp $3=file.txt $#=3
shift
# После shift: $1=/tmp $2=file.txt $#=2

shift 2  # сдвиг на 2 позиции
```

Шаблон разбора опций через `while` + `shift`:

```bash
while [ "$#" -gt 0 ]; do
    case "$1" in
        --outdir) outdir="$2"; shift 2 ;;
        --indir)  indir="$2";  shift 2 ;;
        -*)       echo "Unknown: $1" >&2; exit 1 ;;
        *)        break ;;
    esac
done
# Здесь $@ содержит только оставшиеся файловые аргументы
```

## 9. Конструкция `case`

`case` — аналог `switch` из C/Java, удобен для разбора строк:

```bash
case "$переменная" in
    паттерн1)
        действие1
        ;;
    паттерн2|паттерн3)
        действие2
        ;;
    *)
        действие_по_умолчанию
        ;;
esac
```

Поддерживаются glob-паттерны: `*`, `?`, `[abc]`.

## 10. Код завершения

Каждая команда возвращает **код завершения**:
- `0` — успех
- Ненулевое число — ошибка

```bash
echo $?   # код завершения последней команды

exit 0    # завершить скрипт успешно
exit 1    # завершить скрипт с ошибкой
```

Это важно для использования скрипта в конвейерах и логических выражениях (`&&`, `||`).

## 11. Практические советы

### Всегда заключайте переменные в двойные кавычки

```bash
# Плохо:
mkdir $name         # сломается если name содержит пробел

# Хорошо:
mkdir "$name"
```

### Давайте переменным осмысленные имена

```bash
# Плохо:
tar -czf $1.tar.gz $1

# Хорошо:
source_dir="$1"
tar -czf "${source_dir}.tar.gz" "$source_dir"
```

### Используйте `>&2` для сообщений об ошибках

```bash
echo "Error: file not found" >&2
```

### Используйте `set -e` для остановки при ошибке

```bash
#!/bin/bash
set -e   # скрипт остановится при первой ошибке (ненулевом коде)
set -u   # ошибка при использовании неопределённой переменной
set -o pipefail  # ошибка если любая часть конвейера упала
```

### `getopt` для разбора опций

Для сложных скриптов с множеством опций используйте встроенную команду `getopt`:

```bash
# Описание опций: o=outdir (требует аргумент), i=indir (требует аргумент)
opts=$(getopt -o o:i: --long outdir:,indir: -- "$@")
eval set -- "$opts"
while true; do
    case "$1" in
        -o|--outdir) outdir="$2"; shift 2 ;;
        -i|--indir)  indir="$2";  shift 2 ;;
        --) shift; break ;;
    esac
done
```

## 12. Структура типичного bash-скрипта

```bash
#!/bin/bash
set -euo pipefail

# ---- Константы и значения по умолчанию ----
SCRIPT_NAME="$0"
OUTPUT_DIR="./output"

# ---- Разбор аргументов ----
if [ "$#" -eq 0 ]; then
    echo "Usage: $SCRIPT_NAME [--outdir DIR] <file1> [file2...]" >&2
    exit 1
fi

while [ "$#" -gt 0 ]; do
    case "$1" in
        --outdir) OUTPUT_DIR="$2"; shift 2 ;;
        -*)       echo "Unknown option: $1" >&2; exit 1 ;;
        *)        break ;;
    esac
done

# ---- Основная логика ----
mkdir -p "$OUTPUT_DIR"

for file in "$@"; do
    echo "Processing: $file" >&2
    cp "$file" "$OUTPUT_DIR/"
done

echo "Done. Output in: $OUTPUT_DIR"
```
