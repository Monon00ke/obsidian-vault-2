## 1. Продвинутые возможности sed

### Практический пример: разбивка большого лога

Типичный сценарий применения sed — разбивка большого лога на разделы. Представим скрипт, который последовательно обрабатывает тысячи устройств и пишет в лог:

```
Starting processing device_001
... действия ...
Finished processing device_001
Starting processing device_002
...
```

Механизм парных регулярных выражений в sed идеально подходит для выделения таких блоков:

```bash
sed -n '/^Starting.*device_001/,/^Finished.*device_001/p' log.txt
```

Это ключевая «суперсила» sed: обработка протяжённых блоков с сохранением состояния между строками.

### Краткое резюме команд sed

| Команда | Назначение |
|---|---|
| `p` | Напечатать текущую строку |
| `d` | Удалить строку (не выводить) |
| `s/pat/repl/` | Подстановка (первое вхождение) |
| `s/pat/repl/g` | Подстановка (все вхождения) |
| `a\текст` | Добавить строку **после** текущей |
| `i\текст` | Добавить строку **перед** текущей |
| `c\текст` | Заменить текущую строку целиком |
| `q` | Завершить обработку |
| `n` | Перейти к следующей строке немедленно |

**Пример добавления заготовки docstring ко всем функциям Python:**

```bash
# После каждой строки, начинающейся с "def ", добавить пустой docstring
sed '/^def /a\    """TODO: add docstring"""' script.py
```

### Альтернативный разделитель в команде `s`

Если в шаблоне или замене присутствуют слэши (например, пути файлов), можно использовать любой другой разделитель:

```bash
# Стандартный разделитель — неудобен для путей:
sed 's/\/home\/old/\/home\/new/g' file.txt

# Альтернативный разделитель |
sed 's|/home/old|/home/new|g' file.txt

# Альтернативный разделитель #
sed 's#2026/#2025/#g' file.txt
```

### pattern space и hold space

Внутри sed работают два буфера:

**pattern space** — текущая строка. Именно с ней работают все команды (p, d, s и т.д.). В конце цикла (если нет `-n`) содержимое pattern space выводится.

**hold space** — «постоянная память». Это единственная «переменная» sed. Команды для работы с ней:

| Команда | Действие |
|---|---|
| `h` | Скопировать pattern space → hold space |
| `H` | Добавить pattern space → hold space |
| `g` | Скопировать hold space → pattern space |
| `G` | Добавить hold space → pattern space |
| `x` | Обменять pattern space и hold space |

Hold space позволяет sed «помнить» строки из предыдущих итераций:

```bash
# Напечатать строку перед каждой строкой, содержащей "error"
sed -n '/error/{x;p;x;p}' log.txt
```

**Практическое правило:** если вам понадобился hold space, это чаще всего сигнал перейти на awk.

### Прочие продвинутые возможности sed

- Чтение и запись именованных файлов (команды `r`, `w`)
- Выполнение шелл-команды и подстановка результата
- Метки и переходы (`b метка`, `:метка`)
- Досрочное завершение скрипта (`q`)

## 2. awk

### Почему awk, а не Python?

| Ситуация | Лучший выбор |
|---|---|
| Быстрый однострочник в конвейере | awk |
| Скрипт нужно часто запускать и хранить | Python |
| Нет Python в окружении (контейнер, CI) | sed / awk |
| Обработка табличных данных | awk |
| Сложная логика, классы, исключения | Python |

Python плохо подходит для однострочников из-за синтаксически значимых отступов — вложить два уровня логики в одну строку не получится. В awk это делается естественно.

### Принцип работы awk

awk читает вход **запись за записью** (по умолчанию — строка за строкой) и для каждой записи выполняет все блоки скрипта, чьи условия истинны.

```bash
awk 'условие { действие }' файл
```

Скрипт — это набор пар `условие { действие }`. Если условие опущено — блок выполняется для каждой записи. Если блок опущен — при истинном условии запись выводится.

### BEGIN и END

```bash
awk 'BEGIN { print "Начало" }
     { print NR, $0 }
     END { print "Всего записей:", NR }' файл.txt
```

- **BEGIN** — выполняется один раз до начала чтения данных.
- **END** — выполняется один раз после обработки всех данных.
- Порядок BEGIN/END-блоков между собой имеет значение; обычные блоки — тоже.

### Поля и встроенные переменные

В awk каждая запись автоматически разбивается на **поля** по разделителю (по умолчанию — пробельные символы):

| Переменная | Значение |
|---|---|
| `$0` | Вся текущая запись целиком |
| `$1`, `$2`, ... | Поля 1, 2, ... |
| `$NF` | Последнее поле |
| `NF` | Количество полей в текущей записи |
| `NR` | Номер текущей записи (сквозной) |
| `FS` | Разделитель полей на **входе** (по умолчанию пробел) |
| `RS` | Разделитель записей на **входе** (по умолчанию `\n`) |
| `OFS` | Разделитель полей на **выходе** |
| `ORS` | Разделитель записей на **выходе** |

```bash
# Вывести имена пользователей и их домашние каталоги из /etc/passwd
awk -F: '{ print $1, $6 }' /etc/passwd

# То же через BEGIN
awk 'BEGIN { FS=":" } { print $1, $6 }' /etc/passwd

# Вывести последнее поле каждой строки
awk '{ print $NF }' файл.txt
```

### Переменные и арифметика

awk поддерживает произвольные переменные. Неопределённая переменная равна `0` в числовом контексте и `""` в строковом:

```bash
# Посчитать количество строк (аналог wc -l)
awk 'END { print NR }' файл.txt

# То же через счётчик
awk '{ n++ } END { print n }' файл.txt

# Сумма первого столбца
awk '{ sum += $1 } END { print sum }' файл.txt
```

**Типизация в awk** — слабая (как в JavaScript): тип переменной определяется контекстом использования. Строка `"123x"` в числовом контексте даст `123`.

**Строковая конкатенация** в awk обозначается пробелом (не `+`):

```bash
# "Hello" " " "World" → "Hello World"
awk '{ result = $1 " " $2; print result }'
```

### Условия и циклы

Синтаксис близок к C:

```bash
# if/else
awk '{ if ($1 > 100) print "big"; else print "small" }' файл.txt

# while
awk '{ i=1; while (i <= NF) { print $i; i++ } }' файл.txt

# for
awk '{ for (i=1; i<=NF; i++) print $i }' файл.txt
```

Логические значения: ненулевое число и непустая строка — `true`; `0` и `""` — `false`. Литералов `true`/`false` нет.

### Регулярные выражения в awk

```bash
# Записи, соответствующие регулярке (вся запись)
awk '/error/' лог.txt

# Записи, где второе поле содержит строчную латинскую букву
awk '$2 ~ /[a-z]/' файл.txt

# Записи, где второе поле НЕ содержит цифру
awk '$2 !~ /[0-9]/' файл.txt
```

### Изменение полей и записи

```bash
# Заменить второе поле и вывести всю строку
awk '{ $2 = "ЗАМЕНА"; print }' файл.txt

# Удалить последнее поле
awk '{ NF--; print }' файл.txt

# Поменять поля местами (1 и 3)
awk '{ tmp=$1; $1=$3; $3=tmp; print }' файл.txt
```

### Разделитель записей — абзацы

```bash
# Разбивать на записи по пустым строкам (абзацы)
awk 'BEGIN { RS="" } { print NR": "$0 }' текст.txt
```

### Полезные примеры

```bash
# Напечатать строки с 5 по 10 (аналог sed '5,10p')
awk 'NR>=5 && NR<=10' файл.txt

# Удалить дубликаты строк (сохраняя порядок)
awk '!seen[$0]++' файл.txt

# Сумма значений в каждой колонке CSV (разделитель ;)
awk -F';' '{ for(i=1;i<=NF;i++) sum[i]+=$i }
           END { for(i=1;i<=length(sum);i++) printf "%s ", sum[i]; print "" }' data.csv

# Вывести поля в обратном порядке
awk '{ for(i=NF; i>=1; i--) printf "%s%s", $i, (i>1?OFS:ORS) }' файл.txt

# Аналог wc -l через NR
awk 'END { print NR }' файл.txt
```

### Пользовательские функции (GNU awk)

```bash
awk '
function max(a, b) { return a > b ? a : b }
{ print max($1, $2) }
' файл.txt
```

### Сравнение sed и awk

| | sed | awk |
|---|---|---|
| Единица обработки | Строка | Запись (настраиваемо) |
| Поля | Сложно | Встроено (`$1`, `$2`, ...) |
| Переменные | Hold space (1 штука) | Произвольные |
| Диапазонный адрес `/начало/,/конец/` | Встроен | Нужно реализовывать вручную |
| Сила | Краткость, везде доступен | Табличные данные, логика |
| Шаблонный код | Минимален | Почти нет |

## 3. Пользователи и группы в Linux

### Многопользовательская система

Linux — **многопользовательская** система. Каждое действие выполняется от имени конкретного пользователя. Система может разрешить или запретить действие в зависимости от того, кто его выполняет.

### Пользователи

Каждый пользователь имеет:
- **числовой ID** (UID)
- **символное имя** (login name)
- **домашний каталог**
- **оболочку по умолчанию** (shell)

Специальный пользователь **root** (UID=0) имеет привилегии администратора и может делать практически всё в рамках пользовательского пространства. Однако root — это не то же самое, что уровень ядра ОС.

**Правила хорошего тона:**
- Не работать под root постоянно — только для административных задач.
- Не запускать сервисы (базы данных, веб-серверы) от имени root. Например, PostgreSQL работает под пользователем `postgres`.

### Группы

Пользователь входит в одну или несколько групп. В каждый момент времени у пользователя есть **текущая (первичная) группа**, от имени которой он выполняет действия. Поменять текущую группу: `newgrp имя_группы`.

### Просмотр информации о пользователе

```bash
id                      # UID, GID и все группы текущего пользователя
id имя_пользователя     # то же для другого пользователя
whoami                  # имя текущего пользователя
```

Пример вывода `id`:
```
uid=1000(vasya) gid=1000(vasya) groups=1000(vasya),27(sudo)
```

### Файлы с данными пользователей

| Файл | Содержимое |
|---|---|
| `/etc/passwd` | Имя, UID, GID, домашний каталог, shell. Читают все. |
| `/etc/shadow` | Хэши паролей. Читает только root/shadow. |
| `/etc/group` | Группы и их члены. |

Пример строки из `/etc/passwd`:
```
vasya:x:1000:1000:Vasya Pupkin:/home/vasya:/bin/bash
```
Поля: имя : пароль (x = в shadow) : UID : GID : описание : домашний каталог : shell

### Управление пользователями и группами

| Команда | Назначение |
|---|---|
| `adduser имя` | Создать пользователя (высокоуровневая) |
| `useradd имя` | Создать пользователя (низкоуровневая) |
| `usermod -aG группа имя` | Добавить пользователя в группу |
| `passwd имя` | Сменить пароль |
| `addgroup имя` | Создать группу (высокоуровневая) |
| `groupadd имя` | Создать группу (низкоуровневая) |
| `newgrp группа` | Сменить текущую группу |
| `getent passwd имя` | Получить данные пользователя |

**Важно:** `adduser` / `addgroup` — удобные высокоуровневые обёртки (Debian/Ubuntu). `useradd` / `groupadd` — более низкоуровневые, переносимые.
