# Лекция: Регулярные выражения, grep и sed

---

## 1. Регулярные выражения: теоретическая основа

### Что такое регулярное выражение

Регулярное выражение — формальная система для описания шаблонов строк. Одна и та же система используется в Python, Perl, JavaScript и классических Unix-утилитах (grep, sed, awk).

**Примеры задач, решаемых регулярками:**
- найти все телефонные номера в тексте;
- извлечь IP-адреса, email-адреса, даты;
- найти все шестнадцатеричные константы в исходном коде;
- проверить, соответствует ли строка заданному формату.

### Конечные автоматы

Теоретическая основа регулярных выражений — **конечный автомат**: абстрактное устройство с конечным набором состояний, которое читает символы и переходит из состояния в состояние.

- Есть **начальное** состояние и набор **принимающих** состояний.
- Если автомат, прочитав всю строку, оказался в принимающем состоянии — строка **принята**.
- Для любого регулярного выражения можно построить эквивалентный конечный автомат, и наоборот.

**Что нельзя выразить регулярным выражением:** язык корректных скобочных последовательностей. Для его распознавания нужен счётчик, а конечный автомат не имеет памяти сверх своего состояния.

### Диалекты регулярных выражений

| Режим | Вызов | Особенность |
|---|---|---|
| BRE (basic) | `grep` или `grep -G` | Некоторые спецсимволы требуют экранирования через `\` |
| ERE (extended) | `grep -E` или `egrep` | Спецсимволы работают напрямую; `\(` экранирует скобку |
| PCRE (Perl) | `grep -P` | Расширенный синтаксис, `\d`, `\s` и т.д. |
| fgrep | `grep -F` | Поиск фиксированной строки, без регулярок |

## 2. Синтаксис регулярных выражений

### Обычные символы

Любой не-специальный символ соответствует сам себе:

```
grep "abc" файл.txt     # найти строки, содержащие "abc"
```

### Специальные символы (метасимволы)

| Символ | Значение | Пример |
|---|---|---|
| `.` | Любой один символ | `a.c` → "abc", "a1c", "a c" |
| `?` | Предыдущий элемент 0 или 1 раз | `-?5` → "5" или "-5" |
| `+` | Предыдущий элемент 1 и более раз | `1+2` → "12", "112", "1112" |
| `*` | Предыдущий элемент 0 и более раз | `ab*c` → "ac", "abc", "abbc" |
| `{n}` | Ровно n раз | `.{5}` → любые 5 символов |
| `{n,}` | Не менее n раз | `[0-9]{3,}` → 3+ цифры |
| `{n,m}` | От n до m раз | `[0-9]{2,4}` → 2–4 цифры |
| `^` | Начало строки | `^abc` → строки, начинающиеся с "abc" |
| `$` | Конец строки | `abc$` → строки, заканчивающиеся на "abc" |
| `\` | Экранирование | `\.` → буквальная точка |

### Классы символов `[...]`

```
[abc]        # любой из символов: a, b или c
[a-z]        # любая строчная латинская буква
[0-9]        # любая цифра
[A-Fa-f0-9]  # шестнадцатеричная цифра
[^abc]       # любой символ, кроме a, b, c
```

**Именованные классы** (в квадратных скобках с двоеточием):

```
[[:digit:]]   # цифра (аналог [0-9])
[[:alpha:]]   # буква (учитывает локаль, включая кириллицу)
[[:space:]]   # пробельный символ
[[:alnum:]]   # буква или цифра
```

**Нюанс кириллицы:** диапазон `[а-я]` не включает букву **Ё** (она стоит отдельно в Unicode). Надёжнее использовать `[[:alpha:]]` с нужной локалью или добавлять `ё` явно: `[а-яё]`.

### Группировка и альтернатива

```
(abc)+        # "abc" один и более раз
(0x)?[0-9]+   # необязательный префикс "0x", затем цифры
abc|def       # "abc" или "def"
```

### Обратные ссылки

Каждая пара скобок создаёт **группу**, на которую можно сослаться через `\1`, `\2` и т.д.:

```
# Найти строки, где одно и то же слово встречается дважды
([a-z]+).*\1

# Шестнадцатеричный палиндром: 0x1221
0x([0-9a-f])([0-9a-f])\2\1
```

### Привязки к позиции

```
^[0-9]+$    # строка, состоящая только из цифр
^begin      # строка, начинающаяся со слова "begin"
end$        # строка, заканчивающаяся на "end"
```

### Жадность и откаты

По умолчанию операторы `*`, `+`, `?` — **жадные**: захватывают как можно больше символов. При неудаче происходит **откат** и попытка захватить меньше.

## 3. grep

### Общий вид команды

```bash
grep [опции] 'шаблон' [файл...]
```

Если файл не указан — читает stdin. Это позволяет использовать grep в конвейере.

### Полезные опции

| Опция | Значение |
|---|---|
| `-E` | Расширенный синтаксис ERE (то же, что `egrep`) |
| `-F` | Фиксированная строка, без регулярок (то же, что `fgrep`) |
| `-i` | Игнорировать регистр |
| `-v` | Инвертировать: вывести строки, **не** совпавшие |
| `-n` | Выводить номера строк |
| `-c` | Вывести только количество совпавших строк |
| `-l` | Вывести только имена файлов с совпадениями |
| `-r` | Рекурсивный поиск по каталогу |
| `-o` | Вывести только совпавшие части строк |
| `--` | Отключить интерпретацию опций (если шаблон начинается с `-`) |

### Примеры

```bash
# Найти строки с "hello" в файле
grep "hello" file.txt

# Строки, начинающиеся с цифры
grep "^[0-9]" file.txt

# Шестнадцатеричные числа
grep -E "0x[0-9a-fA-F]+" source.c

# Посчитать количество блоков изменений в выводе diff
diff file1.txt file2.txt | grep -E "^[0-9]+[cda]" | wc -l

# Строки, не содержащие "error"
grep -v "error" log.txt

# Поиск без регулярок (например, если шаблон содержит спецсимволы)
grep -F "a.b.c" file.txt

# Рекурсивный поиск по каталогу
grep -r "TODO" ./src/
```

### Проблема экранирования

В bash некоторые символы интерпретируются **дважды** — сначала bash, потом grep:

```bash
# Плохо: bash «съест» обратный слэш
grep "\.txt" file.txt

# Хорошо: одинарные кавычки отключают интерпретацию bash
grep '\.txt' file.txt

# Четыре слэша, чтобы передать один в регулярку:
# bash: \\\\ → \\, grep: \\ → \
grep '\\' file.txt
```

**Правило:** для шаблонов grep используйте **одинарные кавычки**.

## 4. sed — потоковый редактор

### Зачем sed, если есть Python?

- Python не всегда есть в окружении (Docker-контейнер, CI-система и т.д.).
- sed позволяет писать очень короткие **однострочники** прямо в командной строке.
- Python плохо подходит для однострочников из-за синтаксически значимых отступов.

### Принцип работы

sed читает входной поток **строка за строкой**. Для каждой строки он проходит по всем командам скрипта. Если условие (address) выполнено — команда применяется.

```bash
sed [опции] 'скрипт' [файл...]
```

**Опции:**

| Опция | Значение |
|---|---|
| `-e 'команда'` | Добавить команду (можно несколько `-e`) |
| `-n` | Не выводить строки по умолчанию (только явный вывод через `p`) |
| `-i` | Изменить файл «на месте» (in-place) |

### Структура команды sed

```
[адрес[,адрес]] команда [аргументы]
```

Если адрес не указан — команда применяется ко всем строкам.

### Виды адресов (селекторов)

| Адрес | Значение |
|---|---|
| `5` | Строка номер 5 |
| `$` | Последняя строка |
| `2~5` | Начиная со 2-й, каждая 5-я (GNU sed) |
| `/regex/` | Строки, соответствующие регулярному выражению |
| `2,7` | Строки с 2 по 7 включительно |
| `2,/end/` | С 2-й строки до строки, совпавшей с `/end/` |
| `/begin/,/end/` | С совпадения `/begin/` до совпадения `/end/` |

### Основные команды sed

| Команда | Назначение | Пример |
|---|---|---|
| `p` | Напечатать строку | `sed -n '5p' file` — вывести 5-ю строку |
| `d` | Удалить строку | `sed '3d' file` — удалить 3-ю строку |
| `s/pat/repl/` | Подстановка (первое вхождение) | `sed 's/foo/bar/' file` |
| `s/pat/repl/g` | Подстановка (все вхождения) | `sed 's/foo/bar/g' file` |
| `s/pat/repl/i` | Подстановка без учёта регистра | `sed 's/foo/bar/gi' file` |
| `q` | Завершить обработку | `sed '10q' file` — вывести первые 10 строк |

### Команда подстановки `s` подробнее

```bash
# Заменить первое вхождение "apple" на "orange"
sed 's/apple/orange/' file.txt

# Заменить все вхождения
sed 's/apple/orange/g' file.txt

# Использовать обратные ссылки для перестановки частей даты
# DD.MM.YYYY → MM/DD/YYYY
sed 's/\([0-9]\{2\}\)\.\([0-9]\{2\}\)\.\([0-9]\{4\}\)/\2\/\1\/\3/' file.txt

# В ERE-режиме (GNU sed -E) то же самое проще:
sed -E 's/([0-9]{2})\.([0-9]{2})\.([0-9]{4})/\2\/\1\/\3/' file.txt
```

В строке замены:
- `\1`, `\2`, ... — содержимое группы с соответствующим номером;
- `&` — всё совпавшее выражение целиком.

### Примеры

```bash
# Эмуляция cat (вывод всех строк)
sed -n 'p' file.txt

# Вывести только строки с "error"
sed -n '/error/p' file.txt

# Удалить пустые строки
sed '/^$/d' file.txt

# Вывести строки 20–30
sed -n '20,30p' file.txt

# Вывести блок от "begin" до "end"
sed -n '/begin/,/end/p' file.txt

# Пронумеровать строки (аналог nl)
sed = file.txt | paste - -

# Удалить строки, начинающиеся с "#" (комментарии)
sed '/^#/d' config.txt

# Заменить пробелы на переносы строк (аналог tr)
sed 's/ /\n/g' file.txt
```

### Режим диапазона с состоянием

Когда адрес задан парой, sed «включается» при первом совпадении и «выключается» при втором:

```bash
# Напечатать всё между "begin" и "end" (включительно)
sed -n '/begin/,/end/p' file.txt
```

Если закрывающий адрес так и не встретился — команда работает до конца файла.

## 5. Три утилиты для работы с текстом: сравнение

| | grep | sed | awk |
|---|---|---|---|
| Что делает | Ищет строки по шаблону | Редактирует поток текста | Полноценный язык обработки текста |
| Может изменять текст? | Нет | Да | Да |
| Сложность | Начальная | Средняя | Высокая |
| Типичное применение | Фильтрация, подсчёт | Замена, удаление строк | Табличные данные, вычисления |
