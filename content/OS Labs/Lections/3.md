# Лекция: Стандартные потоки, конвейеры и базовые утилиты

---

## 1. Стандартный вывод (stdout)

### Всё есть файл

В Unix-системе запись в файл и вывод на экран — принципиально одно и то же. Когда Python выполняет `print("hello")`, это вызов `write()` в специальный файловый объект, просто скрытый за кулисами.

При запуске любого приложения командная оболочка автоматически открывает для него два специальных файловых дескриптора:

| Имя | Номер | Назначение |
|---|---|---|
| **stdin** | 0 | Стандартный вход |
| **stdout** | 1 | Стандартный выход |
| **stderr** | 2 | Стандартный вывод ошибок |

По умолчанию stdout и stderr привязаны к терминалу — поэтому текст появляется на экране.

### Перенаправление вывода в файл

Оператор `>` перенаправляет stdout в файл, **уничтожая** прежнее содержимое:

```bash
echo "123" > f1.txt       # создать / перезаписать f1.txt
ls > listing.txt           # сохранить список файлов в файл
```

Оператор `>>` дописывает в конец файла, не стирая старое:

```bash
echo "hello" >> f3.txt    # добавить строку в конец f3.txt
echo "world" >> f3.txt    # ещё одна строка
```

**Важно:** команда `cat f3.txt > f3.txt` уничтожит содержимое файла. Оболочка открывает файл на запись (обрезая его) ещё до того, как `cat` начинает чтение — в итоге файл оказывается пустым.

## 2. Стандартный поток ошибок (stderr)

**stdout** предназначен для основного результата работы программы.  
**stderr** предназначен для всего остального: сообщений об ошибках, предупреждений, прогресса.

Пример: `cat несуществующий_файл` выведет сообщение об ошибке в stderr, а не в stdout. Поэтому при перенаправлении `cat несуществующий_файл > out.txt` сообщение всё равно появится на экране.

### Перенаправление stderr

```bash
# Перенаправить stderr в файл
cat несуществующий_файл 2> errors.txt

# Перенаправить stdout в файл, stderr — туда же, куда stdout
./скрипт > out.txt 2>&1

# Порядок важен! Сначала stdout, потом stderr:
./скрипт > out.txt 2>&1   # правильно: оба потока идут в out.txt
./скрипт 2>&1 > out.txt   # неправильно: stderr пойдёт в терминал
```

Специальный псевдофайл `/dev/null` — «чёрная дыра»: всё записанное туда исчезает:

```bash
./шумный_скрипт > /dev/null    # подавить stdout
./шумный_скрипт 2> /dev/null   # подавить stderr
```

## 3. Стандартный вход (stdin)

### Перенаправление входа из файла

Оператор `<` перенаправляет stdin из файла:

```bash
cat < f5.txt       # cat читает из f5.txt как если бы это был ввод с клавиатуры
wc -l < f5.txt     # посчитать строки в f5.txt
```

Разница между `cat f5.txt` и `cat < f5.txt`:
- `cat f5.txt` — cat сам открывает файл по имени.
- `cat < f5.txt` — оболочка перенаправляет stdin; cat не «знает», что читает файл.

### Псевдофайл `-`

Во многих утилитах `-` означает «читать из stdin». Это позволяет смешивать файлы и stdin:

```bash
cat f1.txt - f2.txt > result.txt
# Сначала выведет f1.txt, затем прочитает ввод с клавиатуры,
# затем выведет f2.txt. Всё запишется в result.txt.
```

## 4. Конвейер (pipe)

Конвейер — главный инструмент Unix. Оператор `|` соединяет stdout одной команды с stdin следующей, без промежуточных файлов:

```bash
ls | wc -l          # посчитать количество файлов в текущем каталоге
cat f.txt | sort | uniq   # отсортировать и убрать дубликаты
```

**Почему конвейер лучше промежуточного файла:**

```bash
# Неправильно — в tmp попадёт и сам tmp-файл:
ls > /tmp/lst.txt
wc -l < /tmp/lst.txt

# Правильно:
ls | wc -l
```

Конвейеры могут быть длинными:

```bash
cat текст.txt | tr ' ' '\n' | sort | uniq -c | sort -rn | head -10
# Список 10 самых частых слов в файле
```

## 5. Логические операторы и статус завершения

### Код возврата процесса

Каждый процесс завершается с **кодом возврата**:
- `0` — успешное завершение.
- Не `0` — ошибка (конкретное значение указывает на вид ошибки).

В Python статус задаётся через `sys.exit(код)`, в C — через возвращаемое значение `main()`.

Проверить код последней команды: `echo $?`

### Последовательное выполнение

```bash
команда1 ; команда2    # выполнить последовательно, независимо от результата
```

### Логическое И (`&&`)

Вторая команда выполняется только если первая завершилась **успешно**:

```bash
make && ./программа    # запустить программу только если сборка прошла успешно
```

### Логическое ИЛИ (`||`)

Вторая команда выполняется только если первая **завершилась с ошибкой** (стратегия fallback):

```bash
команда || echo "Что-то пошло не так"
```

### Пример: диагностика результата

```bash
echo "Запускаем..." >&2
(./do_something && echo "Успех") || echo "Ошибка"
```

## 6. Простые bash-скрипты

Скрипт — файл с последовательностью команд:

```bash
#!/bin/bash
echo "Запускаем..." >&2
pwd
ls
echo "Готово" >&2
```

Чтобы запускать как обычную команду:

```bash
chmod +x мой_скрипт.sh
./мой_скрипт.sh
```

Подавить stdout скрипта:

```bash
./мой_скрипт.sh > /dev/null
```

## 7. Обзор базовых утилит (coreutils)

Все описанные ниже утилиты входят в пакет **coreutils**. Подробная документация: `info coreutils` или `man имя_утилиты`.

**Общий принцип:** если файловый аргумент не указан, утилита читает из stdin. Это позволяет использовать их в конвейерах.

### Работа с содержимым файлов

| Утилита | Назначение | Пример |
|---|---|---|
| `cat` | Вывести содержимое файла(ов) | `cat f1.txt f2.txt` |
| `tac` | Вывести строки в обратном порядке | `tac f.txt` |
| `head` | Первые N строк (по умолчанию 10) | `head -5 f.txt` |
| `tail` | Последние N строк | `tail -20 f.txt` |
| `nl` | Пронумеровать строки | `cat f.txt \| nl` |
| `od` | Дамп бинарного файла | `od -A x -t x1z /bin/cat` |

### Подсчёт

| Утилита | Назначение | Пример |
|---|---|---|
| `wc` | Строки / слова / байты | `wc -l f.txt` |
| `wc -l` | Только строки | `ls \| wc -l` |

### Сортировка и уникальность

| Утилита | Назначение | Пример |
|---|---|---|
| `sort` | Сортировать строки | `sort f.txt` |
| `sort -n` | Числовая сортировка | `sort -n -k2 таблица.txt` |
| `sort -u` | Сортировать и убрать дубликаты | `sort -u f.txt` |
| `uniq` | Убрать соседние дубликаты (нужен предварительный `sort`) | `sort f.txt \| uniq` |
| `shuf` | Перемешать строки случайно | `shuf список.txt` |

### Работа с полями и таблицами

| Утилита | Назначение | Пример |
|---|---|---|
| `cut` | Извлечь поля по разделителю или позиции | `cut -d: -f1,5 /etc/passwd` |
| `paste` | Склеить файлы по столбцам | `paste f1.txt f2.txt` |
| `join` | SQL-подобное соединение двух файлов по общему полю | `join -1 2 -2 3 a.txt b.txt` |
| `tr` | Замена или удаление символов | `tr ' ' '\n'` (слова на отдельных строках) |

### Генерация данных

| Утилита | Назначение | Пример |
|---|---|---|
| `echo` | Вывести строку | `echo "текст"` |
| `printf` | Форматированный вывод | `printf "%d\n" 42` |
| `seq` | Последовательность чисел | `seq 1 10` |
| `yes` | Бесконечно выводить "y" (или другую строку) | `yes \| команда_с_подтверждением` |

### Разное

| Утилита | Назначение | Пример |
|---|---|---|
| `tee` | Одновременно писать в файл и в stdout | `./скрипт \| tee лог.txt` |
| `split` | Разбить файл на части по байтам/строкам | `split -b 1M большой_файл.bin часть_` |
| `cksum` / `md5sum` | Контрольная сумма файла | `md5sum файл.zip` |

### `tee` — смотреть и сохранять одновременно

```bash
./долгий_скрипт | tee лог.txt
# Вывод виден на экране И сохраняется в лог.txt
```

## 8. Три главные утилиты для работы с текстом

### grep — поиск строк по шаблону

**grep** (Global Regular Expression Print) — ищет строки, соответствующие шаблону.

Существует в трёх вариантах:

| Команда | Расшифровка | Особенность |
|---|---|---|
| `fgrep` / `grep -F` | Fixed grep | Ищет буквально, без регулярных выражений |
| `grep` | Basic grep | Базовые регулярные выражения |
| `egrep` / `grep -E` | Extended grep | Расширенные регулярные выражения |
| `rgrep` / `grep -r` | Recursive grep | Поиск во всех файлах каталога |

### sed — потоковый редактор

**sed** (Stream Editor) — обрабатывает текст построчно, позволяя выборочно изменять, удалять или выводить строки. Это более мощный инструмент, чем grep.

### awk — язык обработки текста

**awk** — развитие идей sed, фактически полноценный язык программирования для обработки структурированного текста.

> **grep**, **sed** и **awk** будут разобраны подробно на следующих занятиях.

## Примеры комбинирования утилит

```bash
# Количество файлов в текущем каталоге
ls | wc -l

# Список слов из файла, отсортированный по алфавиту без повторений
cat текст.txt | tr -s ' ' '\n' | sort | uniq

# Имена пользователей и их оболочки из /etc/passwd
cut -d: -f1,7 /etc/passwd

# Пронумеровать строки файла в обратном порядке
tac f.txt | nl

# Посмотреть первые 20 строк hex-дампа бинарника
od -A x -t x1z /bin/ls | head -20

# Показать и сохранить вывод длинной операции
make 2>&1 | tee build.log
```

## Итог лекции

На этой лекции разобраны:
- три стандартных потока: **stdin**, **stdout**, **stderr**;
- перенаправление вывода (`>`, `>>`) и входа (`<`);
- объединение потоков (`2>&1`) и подавление вывода (`/dev/null`);
- **конвейер** (`|`) — соединение утилит без промежуточных файлов;
- **статус завершения** процесса и логические операторы (`&&`, `||`, `;`);
- написание простых **bash-скриптов**;
- обзор утилит пакета **coreutils**: `cat`, `tac`, `head`, `tail`, `wc`, `sort`, `uniq`, `cut`, `paste`, `join`, `tr`, `tee`, `seq`, `yes`, `split`, `nl`, `od`, `shuf`.

На следующем занятии: **grep**, **sed**, регулярные выражения.
