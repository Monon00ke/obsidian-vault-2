## 1. Обработка сигналов в Python: практические примеры

### Установка обработчика и возврат из него

```python
import signal, os

def handler(signum, frame):
    print(f"Got signal {signum}. Ignoring Ctrl+C...")
    # после выхода из обработчика программа продолжает работу

signal.signal(signal.SIGINT, handler)   # перехватить Ctrl+C

while True:
    pass   # основная работа
```

После выхода из обработчика выполнение **продолжается с того места, где было прервано** — если только в обработчике явно не вызван `os._exit()` или `sys.exit()`.

### Graceful shutdown через обработчик

```python
import signal, os, sys

def shutdown(signum, frame):
    print("Shutting down gracefully...")
    # ... завершить задачи, закрыть соединения ...
    os._exit(0)

signal.signal(signal.SIGTERM, shutdown)
signal.signal(signal.SIGINT, shutdown)   # Ctrl+C

while True:
    do_work()
```

### Маскирование и наблюдение за pending-сигналами

```python
import signal, time

# Замаскировать SIGINT (Ctrl+C)
signal.pthread_sigmask(signal.SIG_BLOCK, {signal.SIGINT})

print("SIGINT замаскирован. Нажмите Ctrl+C — сигнал отложится...")
time.sleep(5)

# Посмотреть pending-сигналы (зависшие в ожидании обработки)
pending = signal.sigpending()
print(f"Pending signals: {pending}")

# Снять маску — отложенный SIGINT будет обработан прямо сейчас
signal.pthread_sigmask(signal.SIG_UNBLOCK, {signal.SIGINT})
```

**Важное отличие маскирования от игнорирования:**
- **Маскирование**: сигнал отложен; при снятии маски он будет обработан.
- **Игнорирование**: сигнал немедленно выброшен и забыт навсегда.

### Наследование настроек сигналов через exec

Если перед `execv()` установить игнорирование сигнала, то запущенная программа тоже будет его игнорировать — если не переопределит обработчик явно.

```python
import os, signal

signal.signal(signal.SIGINT, signal.SIG_IGN)   # игнорировать Ctrl+C
os.execv('/usr/bin/sed', ['sed', 's/ab/cd/g'])
# Запущенный sed будет игнорировать Ctrl+C
```

## 2. Сигналы завершения

Всего пять сигналов связаны с завершением процесса:

| Сигнал | Ctrl | Нельзя перехватить | Порождает core dump | Типичное применение |
|---|---|---|---|---|
| **SIGTERM** (15) | — | — | — | Плановое завершение системой/пользователем |
| **SIGINT** (2) | Ctrl+C | — | — | Прерывание пользователем |
| **SIGQUIT** (3) | Ctrl+\\ | — | ✅ | Завершение с дампом для отладки |
| **SIGKILL** (9) | — | ✅ | — | Принудительное уничтожение |
| **SIGABRT** (6) | — | — | ✅ | Самоуничтожение приложения при нарушении инварианта |

### SIGQUIT и core dump

`SIGQUIT` (Ctrl+\\) создаёт **core dump** — бинарный снимок состояния процесса в момент завершения. Отладчик (`gdb`, `lldb`) умеет открывать core dump и показывать стек вызовов, значения переменных и т.д.

Опытные Unix-пользователи используют Ctrl+\\ вместо Ctrl+C, когда хотят разобраться в причине странного поведения. Некоторые приложения перехватывают SIGQUIT и вместо дампа печатают краткую диагностику состояния.

### SIGABRT и `abort()`

`SIGABRT` посылается самому себе функцией `abort()` (в C) — обычно когда нарушен программный инвариант (assert). Порождает core dump. В Python прямого аналога нет (похожая роль у исключений).

## 3. Сигналы остановки

| Сигнал | Ctrl | Нельзя перехватить | Описание |
|---|---|---|---|
| **SIGSTOP** (19) | — | ✅ | Принудительная остановка |
| **SIGTSTP** (20) | Ctrl+Z | — | Вежливая остановка (можно перехватить) |
| **SIGCONT** (18) | — | — | Возобновление остановленного процесса |

### Перехват SIGTSTP для корректной остановки

Если приложение настраивает терминал (например, текстовый редактор), перед остановкой нужно восстановить его в нормальное состояние. Паттерн:

```python
import signal, os

def tstp_handler(signum, frame):
    # Выполнить подготовку к остановке (например, восстановить терминал)
    print("Preparing to suspend...")

    # Снять обработчик, установить дефолтный и самому себе послать сигнал
    signal.signal(signal.SIGTSTP, signal.SIG_DFL)
    os.kill(os.getpid(), signal.SIGTSTP)
    # Здесь процесс остановится

    # При возобновлении (SIGCONT) выполнение продолжится отсюда
    print("Resumed!")
    signal.signal(signal.SIGTSTP, tstp_handler)  # восстановить обработчик

signal.signal(signal.SIGTSTP, tstp_handler)

while True:
    pass
```

### Как работает `fg` в bash

`fg` — это высокоуровневая обёртка: она посылает остановленному процессу (процесс-группе) сигнал `SIGCONT`. Убедиться можно, установив обработчик `SIGCONT`:

```python
def cont_handler(signum, frame):
    print("SIGCONT received (fg was called)")

signal.signal(signal.SIGCONT, cont_handler)
```

## 4. Группы процессов (process groups)

### Что такое группа процессов

**Группа процессов** — набор процессов с одинаковым PGID (process group ID). При `fork()` ребёнок наследует PGID родителя. При `exec()` PGID сохраняется.

PGID всегда равен PID какого-то из процессов группы — **лидера группы**. Если лидер завершился, другие члены группы остаются в ней.

```python
import os

print(f"PID={os.getpid()}, PGID={os.getpgrp()}, SID={os.getsid(0)}")

pid = os.fork()
if pid == 0:
    print(f"Child: PID={os.getpid()}, PGID={os.getpgrp()}, SID={os.getsid(0)}")
    os._exit(0)
```

### Создание новой группы

```python
import os

pid = os.fork()
if pid == 0:
    # Ребёнок отпочковывается в собственную группу
    os.setpgid(0, 0)   # setpgid(pid=0=себя, pgid=0=новая группа со своим PID)
    print(f"Child PGID: {os.getpgrp()}")
    os._exit(0)
```

`os.setpgid(0, 0)` создаёт новую группу, где текущий процесс становится лидером.

### Зачем нужны группы процессов

**Массовая рассылка сигналов:**

```bash
# Послать сигнал всем процессам группы с ID=1234
kill -SIGTERM -1234

# kill с отрицательным числом = сигнал всей группе с этим ID
```

В Python:
```python
import os, signal
os.killpg(pgid, signal.SIGTERM)   # послать сигнал всей группе
```

**В bash:** каждый запущенный pipeline — отдельная группа процессов. `kill %1` посылает сигнал всем процессам этого job (всей группе).

## 5. Сессии (sessions)

### Двухуровневая иерархия

```
Сессия (SID)
  └── Группа процессов A (PGID)
        ├── Процесс 1
        └── Процесс 2
  └── Группа процессов B (PGID)
        ├── Процесс 3
        └── Процесс 4
```

- Процесс принадлежит ровно одной группе.
- Группа целиком принадлежит ровно одной сессии.
- SID = PID процесса, создавшего сессию (лидер сессии).

### Сессия и терминал

SID запущенных процессов совпадает с PID shell, из которого они были запущены. Shell при старте создаёт новую сессию с собой в качестве лидера и привязывает её к терминалу.

```bash
echo $$   # PID текущего shell
```

```python
import os
print(os.getsid(0))   # SID текущего процесса (совпадёт с PID shell)
```

### Создание новой сессии

Процесс может создать новую сессию, если **не является лидером группы**. Это нужно для создания нового shell или daemon-процесса.

```python
os.setsid()   # создать новую сессию; текущий процесс становится её лидером
```

Ограничение «не быть лидером группы» существует, чтобы не разделять группу между двумя сессиями — группа всегда целиком в одной сессии.

### Практическое применение

| Концепция | Shell-применение | Системное применение |
|---|---|---|
| Группа процессов | Pipeline как job | Массовая рассылка сигналов |
| Сессия | Сеанс работы в терминале | Создание daemon без терминала |

**Daemon-процесс** (фоновый сервис без терминала) создаётся через `fork()` → `setsid()` → снова `fork()`. Это стандартный паттерн «двойного форка».

## 6. Bash-функции

### Определение и вызов

```bash
# Определение
function greet() {
    echo "Hello!"
}

# Вызов (без скобок!)
greet
```

### Параметры функции

Параметры передаются при вызове через пробел. Внутри функции — `$1`, `$2`, ..., `$@`, `$#`:

```bash
function add() {
    local result=$(expr $1 + $2)
    echo $result
}

# Вызов
add 3 5   # выведет: 8
```

### Возвращаемые значения

В bash есть **два способа** вернуть значение:

**1. Числовой код возврата (0–255) через `return`:**

```bash
function check_positive() {
    if [ "$1" -gt 0 ]; then
        return 0   # успех
    else
        return 1   # ошибка
    fi
}

check_positive 5
echo $?   # 0 (успех)

check_positive -3
echo $?   # 1 (ошибка)
```

Ограничение: только числа от 0 до 255. Число вне диапазона берётся по модулю 256.

**2. Строковое значение через stdout:**

```bash
function get_name() {
    echo "Alice"
}

# Захватить stdout функции
name=$(get_name)
echo "Name: $name"   # Name: Alice

# Или через обратные кавычки (backticks)
name=`get_name`
```

### Локальные переменные

Без `local` переменная становится глобальной:

```bash
function f() {
    local x=10   # локальная
    y=20         # глобальная!
}

f
echo $x   # пусто
echo $y   # 20
```

### Арифметика в bash

Bash не имеет встроенных арифметических выражений. Для вычислений используют:

```bash
# Вариант 1: утилита expr
result=$(expr 3 + 5)   # 8

# Вариант 2: двойные скобки (предпочтительнее)
result=$(( 3 + 5 ))    # 8
result=$(( $a * $b ))  # перемножить переменные
```

### Обработка сигналов в bash через `trap`

```bash
# Установить обработчик сигнала в bash
trap 'функция_или_команда' СИГНАЛ

# Примеры:
trap cleanup SIGTERM SIGINT   # вызвать функцию cleanup
trap '' SIGINT                # игнорировать Ctrl+C
trap - SIGINT                 # восстановить обработку по умолчанию

function cleanup() {
    echo "Cleaning up..."
    # ... завершающие действия ...
    exit 0
}

trap cleanup SIGTERM SIGINT

while true; do
    sleep 1
done
```

### Пример: скрипт с graceful shutdown

```bash
#!/bin/bash

RUNNING=true

function on_exit() {
    echo "Signal received, shutting down..."
    RUNNING=false
}

trap on_exit SIGTERM SIGINT SIGHUP

while $RUNNING; do
    echo "Working..."
    sleep 2
done

echo "Done."
```
